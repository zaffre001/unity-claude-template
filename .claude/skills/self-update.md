# Skill: /self-update (Knowledge Synthesis Instruction)

당신은 프로젝트의 '지식 관리자'입니다.
지침이 너무 비대해지지 않도록 주의하며,
습득한 정보를 적절한 계층으로 분류하십시오.

### 1. 지식의 분류 원칙
- **범용 레이어:** 어느 Unity 프로젝트에나 적용되는 재사용 가능한 지식.
- **도메인 레이어:** 이 프로젝트만의 고유한 규칙과 비즈니스 로직.
- **실시간 레이어:** 현재 태스크의 의존성과 휘발성 컨텍스트.

### 2. 학습 프로세스
- **지침 최적화:** 새로운 지식 추가 시 기존 지침이 너무 커진다면
              계층 분리를 제안하십시오.
- **원리 중심:** 단순 코드 조각이 아닌 '왜(Why)'에 집중하여 정리하십시오.
- **직접 수정 금지:** diff 형식으로 제안하고 아키텍트의 승인을 기다리십시오.

### 3. 반영 대상 판단 기준

이 내용이 위반될 경우 시스템이 실제로 망가지는가?
  YES → RULES.md

  NO → 이 내용이 매 세션 바뀌는 실시간 정보인가?
    YES → CLAUDE.local.md (하네스 자동 주입, gitignore 권장)

    NO → 이 내용이 특정 작업 타입에서만 필요한가?
      YES → .claude/skills/{skill}.md
      NO  → CLAUDE.md

### 4. 출력 형식

```markdown
## /self-update 결과

### 이번 세션에서 새로 습득한 지식
{요약}

### 제안하는 변경
**대상 파일:** {CLAUDE.md | RULES.md | .claude/skills/*.md}
**섹션:** {섹션명}
**변경 유형:** {추가 | 수정 | 삭제}

\```diff
+ 추가할 내용
- 삭제할 내용
\```

**이유:** {왜 이 지침이 필요한지}

---
승인하시면 파일을 직접 수정하겠습니다.
```
