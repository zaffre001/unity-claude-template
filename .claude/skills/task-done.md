# Skill: /task-done (Task Completion)

코드 수정이 끝나면 반드시 이 마무리를 수행하십시오.
코드를 수정하고 세션을 끝내는 것은 반쪽짜리 작업입니다.

### STEP 1 — 변경 사항 요약
- 수정한 파일, 변경한 심볼, 추가/삭제 라인 수를 나열한다.
- 의도한 변경과 실제 변경이 일치하는지 확인한다.
- **의도하지 않은 변경이 있으면 즉시 보고한다.**

### STEP 2 — 자체 검증
다음 항목을 하나씩 확인한다:
- [ ] 신규 이벤트 구독이 있다면 → `OnDestroy()`에서 해제 쌍이 존재하는가
- [ ] `async` 메서드가 있다면 → `CancellationToken`이 전달되는가
- [ ] `static` 변수를 추가했다면 → `[RuntimeInitializeOnLoadMethod]`로 초기화하는가
- [ ] 에디터 전용 코드가 있다면 → `#if UNITY_EDITOR`로 감싸져 있는가
- [ ] RULES.md 전 규칙 위반이 없는가

### STEP 3 — Lock 해제
- STEP 4(/task-start)에서 Lock을 획득했다면 반드시 해제한다.
- 하네스가 없으면 이 단계를 건너뛴다.

### STEP 4 — worktree 정리
- 워크트리에서 작업한 경우, 머지 상태를 확인한다.
- 아키텍트 리뷰 후 정리 예정이면 그대로 둔다.

### STEP 5 — /self-update 판단
다음 조건 중 하나라도 충족되면 `/self-update`를 실행한다:
- 기존 지침에 없는 새로운 패턴을 발견했다
- 기존 지침이 부정확하거나 불완전함을 확인했다
- 반복될 가능성이 높은 실수를 했다

### STEP 6 — 완료 보고
작업 결과를 간결하게 보고한다.

### 출력 형식

```
[STEP 1] 변경 사항 요약
수정한 파일:
- {파일}
  변경: {요약}
  라인: +{N} / -{N}
의도하지 않은 변경: {없음 | 설명}

[STEP 2] 자체 검증
[✓] 항목 1
[✓] 항목 2
...

[STEP 3] Lock 해제
{상태}

[STEP 4] worktree 정리
{상태}

[STEP 5] /self-update 판단
{실행 여부와 사유}

[STEP 6] 완료 보고
{한 줄 요약}
```
