# Skill: /task-start (Task Briefing)

작업을 시작하기 전에 반드시 이 브리핑을 수행하십시오.
파일을 열기 전에 범위를 먼저 확정합니다.

### STEP 1 — RULES.md 재확인
- RULES.md를 읽고, 이번 작업과 관련된 규칙을 특정한다.
- 위반 위험이 있는 규칙이 있다면 명시적으로 선언한다.

### STEP 2 — 대상 파악
- `grep`으로 실제 파일 위치를 확인한다. **추측하지 않는다.**
- 대화 내역에서 경로를 꺼내지 않는다. 반드시 직접 조회한다.
- 관련 파일과 심볼을 모두 나열한다.

```bash
# 예시
grep -r "class TargetClass" Assets/ --include="*.cs" -l
grep -r "TargetMethod" Assets/ --include="*.cs" -l
```

### STEP 3 — 작업 범위 선언
- 수정할 파일과 심볼을 명시한다.
- 수정하지 않는 파일도 명시한다 (호출부 확인만 등).
- 범위 밖의 작업이 발견되면 아키텍트에게 보고한다.

### STEP 4 — Lock 획득
- 하네스 세션이 활성화되어 있고 SymbolLock 시스템이 있다면 Lock을 획득한다.
- 하네스가 없으면 이 단계를 건너뛴다.

### STEP 5 — 작업 시작 선언
- 모든 사전 조건이 확인되었음을 선언한다.
- 작업 내용을 한 줄로 요약한다.

### 출력 형식

```
[STEP 1] 관련 규칙 확인
{규칙 목록과 위반 위험 여부}

[STEP 2] 대상 파악
{grep 결과와 파일 위치}

[STEP 3] 작업 범위 선언
수정할 파일:
- {파일} → {심볼}
수정하지 않는 파일:
- {파일} (사유)

[STEP 4] Lock 획득
{Lock 상태 또는 "하네스 비활성 — 건너뜀"}

[STEP 5] 작업 시작
{한 줄 요약}
```
